# ПАРСЕР СЛОВАРЯ РУССКОГО ЯЗЫКА XVIII ВЕКА

**Цель проекта:** преобразование HTML файлов, содержащих словарные статьи в структурированный JSON с выделением лексикографических зон.

**Исходные данные:** выпуски словаря русского языка XVIII века в формате HTML. Каждый выпуск содержит несколько тысяч словарных статей.

---

## DICTIONARY_PIPELINE.PY

Скрипт представляет собой комплексный пайплайн для автоматического парсинга HTML-файлов словарных статей. Он извлекает информацию из HTML-файлов и сохраняет её в формате JSON. 

### Структура скрипта
```text
dictionary_pipeline.py
├── Конфигурационные константы
├── HTML-парсер (RichHTMLParser) — разбор форматирования (bold, italic)
├── Функции обработки текста 
├── Извлечение информации из заголовков
├── Разбор определений и значений
├── Выделение грамматических помет
├── Обработка смысловых кластеров
├── Внешние модули (цитаты, коллокации, этимология)
└── Основной пайплайн (build_article_json) — сборка JSON
```

### Конфигурационные константы
* **PROJECT_ROOT** — корневая директория проекта.
* **DEFAULT_INPUT_DIR** — папка с входными HTML-файлами. 
* **DEFAULT_OUTPUT_DIR** — папка для выходных JSON-файлов.

---

## Внешние модули

Скрипт динамически подключает специализированные модули для глубокого анализа лексикографических зон:

* **separate_citations.py (almighty_architect)**: Отделение цитат от источников. Использует внешний DOCX-указатель для точной идентификации шифров, очищает текст от технических маркеров.
* **CollocationPhaseo.py (As_hen_ok)**: Извлечение устойчивых сочетаний (коллокаций) по символьным маркерам (`~`, `⸢SYM_COLLOCATION⸣`).
* **find_etymologies_in_pipeline.py (pedrobirq)**: Извлечение этимологических сведений. Ищет данные в квадратных скобках, распознает языковые маркеры (*лат., фр.*) и цепочки заимствований.
* **dynamic_tags.py (ProdamGarageXeX)**: Извлечение динамических помет. Выделяет теги, описывающие исторические изменения слова (`SYM_NEW_WORD`, `SYM_OBSOLETE`).

## Способы запуска

### Зависимости
* **Python 3.8+**
* **Библиотеки**: `BeautifulSoup4`, `python-docx` (для модуля цитат).
* **Стандартные**: `html`, `json`, `re`, `pathlib`, `argparse`, `functools`, `glob`, `tkinter`, `threading`.

### Варианты запуска
1. **Графический интерфейс (GUI):**
   Запустите `gui.py`. При запуске откроется окно с предустановленными путями из конфигурации. Вы сможете отслеживать прогресс обработки визуально.
   ```bash
   python gui.py
   ```
   *[Подробная документация](scripts/ulia_pav/README.md)*

2. **Командная строка:**
   Используйте `dictionary_pipeline.py` для пакетной обработки через аргументы:
   ```bash
   python dictionary_pipeline.py --source "./path/to/html" --output "./path/to/json"
   ```
