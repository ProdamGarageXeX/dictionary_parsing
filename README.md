# Парсер Словаря русского языка XVIII века

Комплексное программное решение для преобразования лексикографических данных из формата HTML в структурированный JSON. Проект реализует автоматическое выделение зон словарной статьи: от заголовочных слов до этимологии и цитат.

---

## 🛠 Основной пайплайн (`dictionary_pipeline.py`)

Скрипт-оркестратор координирует работу всех подмодулей и выполняет сборку финального JSON-объекта.

### Архитектура парсера:
* **RichHTMLParser**: Глубокий разбор HTML с сохранением семантического форматирования (*bold*, *italic*, *sup*).
* **Header Processor**: Извлечение заголовочных слов и работа с надстрочными индексами (омонимия).
* **Definition Engine**: Автоматическое разделение кластеров значений (римские/арабские цифры).
* **Grammar Analyzer**: Выделение грамматических и стилистических помет из специализированного списка (более 70 тегов).
* **External Integration**: Сбор данных из внешних модулей (этимология, цитаты, коллокации).

## 🧩 Интегрированные внешние модули

Проект объединяет наработки нескольких разработчиков для обеспечения максимальной точности парсинга:

| Модуль | Автор | Назначение |
| :--- | :--- | :--- |
| `separate_citations.py` | almighty_architect | Сверяет текст с указателем источников и отделяет цитату от шифра. |
| `CollocationPhaseo.py` | As_hen_ok | Находит фразеологизмы и коллокации по маркерам (`~`, `⸢SYM...⸣`). |
| `find_etymologies.py` | pedrobirq | Извлекает сведения о происхождении слова (языки, цепочки заимствований). |
| `dynamic_tags.py` | ProdamGarageXeX | Фиксирует исторические пометы (*новое слово, выпало из употр.*). |
| `extract_sup_articles.py`| ulia_pav | Обрабатывает надстрочные индексы для различения омонимов. |

## 🚀 Запуск и использование

### Зависимости
* **Python 3.8+**
* **Библиотеки**: `BeautifulSoup4`, `python-docx` (для чтения справочников).
* Стандартные модули: `re`, `json`, `pathlib`, `tkinter`, `threading`.

### Способы запуска
1.  **Графический интерфейс (GUI)**:
    Запустите `gui.py`. В окне будут предустановлены пути к данным. Вы сможете отслеживать прогресс обработки визуально.
    ```bash
    python gui.py
    ```

2.  **Командная строка**:
    Используйте `dictionary_pipeline.py` для пакетной обработки через аргументы `--source` и `--output`.
